About Fubar Analytics

A personal sandbox for hacking around with long-run financial market data. Not a product, not a service, not financial advice — just a place to run pencil-on-paper questions through real historical data and publish what comes out.

What this site is

Every analysis here takes a contribution schedule (a lump sum, a recurring deposit, eventually a withdrawal plan or a factor tilt) and applies it to every rolling N-year window in U.S. equity history since July 1926. Roughly a thousand 10-year windows, each a small history lesson. The point isn't a single backtest with a single answer; it's the shape of the outcomes — distributions, dispersion, how often the strategy lost money in real terms, which decades were brutal and which were a gift.

The math runs offline in Python; the site is a thin static layer that visualizes the results. Charts are interactive. Every scenario also ships a downloadable PDF report you can save or send to someone.

Why it exists

Most "what would $X have grown to?" calculators give you one number — the outcome for one starting month, usually whichever one makes the strategy look best. That hides almost everything interesting about long-run returns: how much the starting decade matters, how wide the dispersion is, what the worst case looks like. Rolling windows surface that. The same $1,000 lump sum invested in January 1990 vs. January 2000 vs. October 1929 tells you radically different stories about the market — and looking at all the starting months together is the closest you get to seeing the market distribution, not just one sample of it.

Beyond that, the site is just a place to mess around. New scenario ideas come up; the registry pattern makes it easy to add one and ship.

Data sources

  • Ken French Data Library — value-weighted U.S. total-market returns, July 1926 onward, derived from CRSP. The academic gold standard for U.S. equity research.
  • FRED CPIAUCNS — monthly U.S. CPI, not seasonally adjusted, for converting nominal returns into real (inflation-adjusted) returns via the Fisher equation. The not-seasonally-adjusted series is used deliberately so the data extends back to 1913 and aligns with Ken French's 1926 start.

Both sources are free, public, and well-documented. The full Python pipeline is in the source repository on GitHub if you want to see exactly how the data gets from those sources to the charts.

How it works

Two deliberately separated layers:

  • Offline (Python). src/ingest.py downloads source data and writes a tidy monthly returns Parquet file. src/scenarios/ declares each scenario as a module. src/export.py runs every scenario over every horizon and emits a JSON file per scenario. src/report.py builds a per-scenario PDF report using a shared template.
  • Online (browser). A small static site (no build step, no bundler, no Node modules) loads the JSON for the current scenario and renders interactive Plotly charts. GitHub Pages serves the whole thing directly from the docs/ folder.

The split is deliberate: pandas is enormously better than vanilla JS for time-series math, but the JS side stays small, fast, and free to host.

Important caveats

This site is research output, not financial advice. Past performance doesn't predict future results. Every scenario fixes specific assumptions — no taxes, no fees, no rebalancing, U.S. market only, monthly granularity — and your real-life experience would differ from any of them in ways that matter. The goal is to make the shape of historical outcomes intuitive, not to back-test a strategy to put money into.

If you're trying to decide what to do with actual money, this site is one of many inputs at best — and there are professionals whose job it is to weigh those inputs for your specific situation.

Colophon

Built with Python (pandas, matplotlib), Plotly.js, and Pico CSS. Hosted on GitHub Pages. The site has no analytics, no cookies, no tracking, no JavaScript beyond what's needed to fetch the JSON and draw the charts. Source on GitHub.
